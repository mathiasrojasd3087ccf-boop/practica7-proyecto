# Informe Práctica 7
# Práctica 7 - Restricciones, objetivos y alcance del proyecto

Identificación de restricciones del proyecto
- Falta de tiempo para desarrollar todas las prácticas.  
- Conocimiento limitado de Git y herramientas de programación.  
- Recursos tecnológicos limitados (computadora, internet).  
- Entorno académico con cargas de otros cursos.  

Propuesta de alternativas de solución
- Planificación semanal para avanzar prácticas poco a poco.  
- Uso de tutoriales y guías paso a paso para aprender Git.  
- Uso de repositorios individuales por práctica para mantener el orden.  
- Priorizar prácticas más complejas y organizarlas por dificultad.  

Objetivos del proyecto

Objetivo general
Organizar y evidenciar todas las prácticas de campo de manera individual usando Git y GitHub, asegurando calidad y control de versiones.  

Objetivos específicos
1. Crear repositorios separados para cada práctica.  
2. Documentar correctamente cada práctica con informe y README.  
3. Implementar control de versiones con ramas y commits claros.  
4. Subir todo a GitHub para generar evidencia accesible al profesor.  

Alcance del proyecto
- Se incluirán **todas las prácticas de campo 1 a 13**.  
- Cada práctica tendrá su repositorio con README e informe.  
- Se evidenciará el uso de Git, GitHub, ramas, commits y documentación.  
- No incluye colaboración con compañeros ni revisión de Pull Request (ya que se hace de manera individual).
